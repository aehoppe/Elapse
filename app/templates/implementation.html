The Making of Elapse.

This project was developed with Python, HTML, CSS, and a little bit of JavaScript. Our source code can be accessed in our Github repository. The repository’s README contains all necessary information for running Elapse locally, including all dependency information.


First, an overview of the user-software interactions that occur when using Elapse:





The user starts the interaction by going to elapseapp.herokuapp.com and selecting Use Elapse from the homepage, or simply clicking on the header/image, as prompted by the hover effects. The user is then prompted to upload their calendar data as a .ics file and select a week to visualize. The ical is received and parsed for events within the selected date range, and these events are stored within our own data structure. These events are then handed back to the user on a webpage, where the user verifies the data is as intended or can choose to select a different file or week. The user is prompted to select a visualization mode, and their calendar information is then visualized on a webpage.


Our backend development was done in Python, and makes use of the Flask framework for webification. The users’ upload form was created with flask_wtf. We parse users’ ical (.ics) files using the icalendar library, and store the data as elapseEvent objects within an elapseCalendar object. We use the vincent library to visualize our data. Vincent is a wrapper for vega, which is a wrapper for d3.js. We had initially hoped to simply do the visualization and all data processing in JavaScript, but decided to find a Python alternative so that we could focus on improving our Python skills as opposed to working in an entirely unfamiliar language. With vincent, we generate a JSON object of the visualization, which is then rendered in an HTML template.





On the frontend, we use JavaScript and jQuery to render the visualization JSON object with the HTML template, as well as to generate the user’s event-verification interface. For the website design, we use the Bootstrap framework, and employ CSS flexboxes in a few cases. The webapp is hosted on Heroku.


Throughout the project, we both reviewed our code with peers and sought feedback from peers regarding design decisions we faced. Our code reviewers helped us to realize how important it was for our code to be modular (which we would have probably realized eventually, but they brought it to the foreground), and group ideation sessions with peers helped us to think critically about the types of user interactions we wanted to foster. Documentation from these reviews can be found in the documentation directory of our Github repository.
